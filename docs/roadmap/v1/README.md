# Api.SystemDefinitions Roadmap (v1)

## Vision

Provide a small, deterministic platform-definition library for applications that
need to map game system names, aliases, folders, emulator cores, extensions, and
IGDB identifiers.

## Current Status

- [x] In progress
- [ ] Stable
- [ ] Publication-ready

## Goals

- [x] Load system definitions from a local fixture or remote dataset.
- [x] Normalize common system aliases.
- [x] Keep default tests deterministic and fixture-backed.
- [x] Package README and license metadata.
- [x] Add CI build/test/pack workflow.
- [x] Add automated dataset refresh tooling.
- [ ] Add NuGet publish workflow after release credentials are confirmed.

## Known Gaps

- [ ] No generated API reference yet.
- [x] No fixture refresh script yet.
- [ ] Publish remains manual.
